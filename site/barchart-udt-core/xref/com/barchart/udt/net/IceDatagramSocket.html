75  	/**
76  	 * Connects the socket to a remote address for this socket. When a socket is
77  	 * connected to a remote address, packets may only be sent to or received
78  	 * from that address. By default a datagram socket is not connected.
79  	 * 
80  	 * <p>
81  	 * If the remote destination to which the socket is connected does not
82  	 * exist, or is otherwise unreachable, and if an ICMP destination
83  	 * unreachable packet has been received for that address, then a subsequent
84  	 * call to send or receive may throw a PortUnreachableException. Note, there
85  	 * is no guarantee that the exception will be thrown.
86  	 * 
87  	 * <p>
88  	 * A caller's permission to send and receive datagrams to a given host and
89  	 * port are checked at connect time. When a socket is connected, receive and
90  	 * send <b>will not perform any security checks</b> on incoming and outgoing
91  	 * packets, other than matching the packet's and the socket's address and
92  	 * port. On a send operation, if the packet's address is set and the
93  	 * packet's address and the socket's address do not match, an
94  	 * IllegalArgumentException will be thrown. A socket connected to a
95  	 * multicast address may only be used to send packets.
96  	 * 
97  	 * @param address
98  	 *            the remote address for the socket
99  	 * 
100 	 * @param port
101 	 *            the remote port for the socket.
102 	 * 
103 	 * @exception IllegalArgumentException
104 	 *                if the address is null, or the port is out of range.
105 	 * 
106 	 * @exception SecurityException
107 	 *                if the caller is not allowed to send datagrams to and
108 	 *                receive datagrams from the address and port.
109 	 * 
110 	 * @see #disconnect
111 	 * @see #send
112 	 * @see #receive
113 	 */

493 	 * Sets traffic class or type-of-service octet in the IP datagram header for
494 	 * datagrams sent from this DatagramSocket. As the underlying network
495 	 * implementation may ignore this value applications should consider it a
496 	 * hint.
497 	 * 
498 	 * <P>
499 	 * The tc <B>must</B> be in the range <code> 0 <= tc <=
500 	 * 255</code> or an IllegalArgumentException will be thrown.
501 	 * <p>
502 	 * Notes:
503 	 * <p>
504 	 * for Internet Protocol v4 the value consists of an octet with precedence
505 	 * and TOS fields as detailed in RFC 1349. The TOS field is bitset created
506 	 * by bitwise-or'ing values such the following :-
507 	 * <p>
508 	 * <UL>
509 	 * <LI><CODE>IPTOS_LOWCOST (0x02)</CODE></LI>
510 	 * <LI><CODE>IPTOS_RELIABILITY (0x04)</CODE></LI>
511 	 * <LI><CODE>IPTOS_THROUGHPUT (0x08)</CODE></LI>
512 	 * <LI><CODE>IPTOS_LOWDELAY (0x10)</CODE></LI>
513 	 * </UL>
514 	 * The last low order bit is always ignored as this corresponds to the MBZ
515 	 * (must be zero) bit.
516 	 * <p>
517 	 * Setting bits in the precedence field may result in a SocketException
518 	 * indicating that the operation is not permitted.
519 	 * <p>
520 	 * for Internet Protocol v6 <code>tc</code> is the value that would be
521 	 * placed into the sin6_flowinfo field of the IP header.
522 	 * 
523 	 * @param tc
524 	 *            an <code>int</code> value for the bitset.
525 	 * @throws SocketException
526 	 *             if there is an error setting the traffic class or
527 	 *             type-of-service
528 	 * @since 1.4
529 	 * @see #getTrafficClass
530 	 */
